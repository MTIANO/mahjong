# MTiano - 日麻工具集合小程序设计文档

## 概述

日麻（日本麻将）工具集合微信小程序，提供番型表/点数表快速查询和拍照识别麻将牌自动算番功能。

## 技术栈

- **前端**：原生微信小程序
- **后端**：Go + Gin
- **数据库**：MySQL
- **图像识别**：AI 视觉 API（具体服务待定，通过接口抽象）

## 架构设计

采用**静态数据前置**方案：

- 番型表和点数表作为静态 JSON 打包在小程序端，查询纯本地
- 后端专注于图像识别 + 算番计算

```
小程序 (静态查询 + 拍照上传)
        │ HTTPS
Go 后端 (图片接收 → AI识别 → 算番引擎 → 返回结果)
        │
     MySQL (用户数据/历史记录)
```

## 小程序前端

### 页面结构

| 页面 | 功能 |
|------|------|
| `pages/index` | 首页导航 |
| `pages/yaku` | 番型表查询（搜索/分类筛选） |
| `pages/score` | 点数表查询（番数×符数 → 得失点） |
| `pages/camera` | 拍照识别入口 |
| `pages/history` | 识别历史记录 |

### 静态数据

- `data/yaku.json` — 番型列表（名称、番数、描述、示例、开闭门区分）
- `data/score.json` — 点数对照表（番数×符数 → 亲/子得失点）

### 拍照识别流程

1. 用户拍照或从相册选择图片
2. 上传图片到后端 `POST /api/v1/recognize`
3. 后端返回识别结果
4. 前端展示：识别出的牌面 + 番型列表 + 总番数 + 得分

### API

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/v1/recognize` | 上传图片 → 返回识别结果+番型+得分 |
| GET | `/api/v1/health` | 健康检查 |

### 算番引擎

- **输入**：14 张牌（或带副露的手牌）+ 和牌条件（自摸/荣和、场风、自风等）
- **处理**：拆解手牌 → 枚举和牌形式 → 判定适用番型 → 取最优解
- **输出**：番型列表 + 总番数 + 符数 + 亲/子得失点

### AI 视觉接口抽象

```go
type VisionService interface {
    RecognizeTiles(ctx context.Context, image []byte) ([]Tile, error)
}
```

通过接口抽象实现 AI 服务可替换。

## 数据模型

### 麻将牌 (Tile)

- 万子 (Man): 1m-9m
- 筒子 (Pin): 1p-9p
- 索子 (Sou): 1s-9s
- 字牌 (Honor): 东南西北白发中

### 番型 (Yaku)

- 名称（日文/中文）
- 番数（开门/闭门）
- 判定条件

## 第一版 MVP 范围

1. 番型表查询（静态数据本地查询）
2. 点数表查询（番数×符数快速计算）
3. 拍照识别 → 后端处理 → 返回结果
4. 算番引擎核心逻辑

暂不包含：用户系统、历史记录持久化、社交功能。
